# IXO Engine

IXO Engine은 엔트리 스타일의 노드 기반 로직과 Canvas Builder를 결합한 비주얼 앱 제작 엔진입니다. 사용자는 노드를 연결해 실행 흐름을 만들고, Canvas Builder에서 실제 사용자 인터페이스를 배치한 뒤 Windows와 Linux용 산출물로 내보낼 수 있습니다.

현재 문서는 **V1.1.3** 기준입니다.

## 핵심 철학

- 로직은 노드 그래프로 표현합니다.
- 화면은 Canvas Builder에서 시각적으로 설계합니다.
- UI 요소와 노드는 Ref Key, Scene, Binding 정보를 통해 함께 동기화됩니다.
- exported runtime은 편집기와 동일한 프로젝트 데이터를 사용해 결과 차이를 줄입니다.
- 네트워크, 파일 감시, 스크립트처럼 위험할 수 있는 기능은 사용자 동의와 권한 정책을 우선합니다.

## 주요 작업 영역

| 영역 | 설명 |
|---|---|
| Node Workspace | 노드를 배치하고 연결해 실행 흐름을 만듭니다. |
| Canvas Builder | 텍스트, 이미지, 입력창, 버튼, 컨테이너, Vector 요소를 직접 배치하고 수정합니다. |
| UI Viewer | 제작 중인 화면을 사용자 관점에서 미리 확인합니다. |
| Inspector | 선택한 노드 또는 UI 요소의 값을 편집합니다. V1.1.3부터 기본 UI는 단순화되어 있으며, 설정에서 고급 모드를 켤 수 있습니다. |
| Functions | 사용자가 직접 함수 그래프를 만들고, 함수 호출 노드 또는 액션에서 실행합니다. |
| Scene Manager | 여러 화면을 Scene 단위로 관리합니다. |
| Asset Manager | 이미지, 사운드, 아이콘, `.ixo` 프로젝트 조각, 테마 파일을 관리합니다. |
| Error Log Console | 실행 결과, 분기 결과, 외부 액션 로그를 확인합니다. 기본값은 꺼져 있으며 설정에서 표시할 수 있습니다. |
| Settings | 네트워크 동의서, 로그 표시, 고급 모드, 노드 연결점 크기, 테마, 내보내기 기본값 등을 관리합니다. |

## V1.1.3 주요 변경사항

- 업데이트 적용 버튼을 누르면 파일을 다운로드 폴더에 따로 저장하는 방식이 아니라, 앱 내부에서 즉시 업데이트 적용 흐름으로 이동합니다.
- `F5`를 누르면 File 내보내기를 하지 않아도 별도 창으로 앱 미리보기를 실행할 수 있습니다.
- 문서 버튼을 눌러도 문서가 열리지 않던 문제를 수정했습니다.
- Canvas Builder에서 UI 요소를 클릭해도 선택 또는 속성 수정이 되지 않던 문제를 수정했습니다.
- 텍스트 입력창에 입력이 반영되지 않던 문제를 수정했습니다.
- 노드 검색과 `Ctrl+F` 검색 포커스 문제가 수정되었습니다.
- 속성 창을 단순화했습니다. 기존처럼 자세한 항목을 보고 싶다면 설정에서 `고급 모드 활성화`를 켜면 됩니다.
- 노드 연결점 크기를 설정에서 조절할 수 있습니다.
- 로그 패널 표시 여부를 설정에서 켜고 끌 수 있습니다.
- 네트워크 노드 이용 동의서를 Windows 기본 창이 아닌 IXO Engine 내부 UI로 표시합니다.
- 설정의 네트워크 항목에서 동의서를 다시 볼 수 있고, 동의를 취소할 수 있습니다.
- Scene, Canvas Builder, Settings, Function Editor가 브라우저형 탭으로 열립니다.
- 탭은 닫기 버튼을 제공하며, Scene 탭은 우클릭으로 이름을 바꿀 수 있습니다.
- 탭끼리 드래그해 위치를 바꿀 수 있습니다.
- Canvas Builder의 반응형 미리보기 가로/세로 크기를 조절할 수 있습니다.
- 전역변수와 지역변수를 추가, 수정, 삭제할 수 있습니다.
- 편집 중인 함수도 탭으로 열려 현재 작업 위치를 더 쉽게 파악할 수 있습니다.

## 브라우저형 탭 작업 방식

V1.1.3부터 Scene, Canvas Builder, Settings, Function Editor가 상단 탭에서 관리됩니다.

| 탭 종류 | 동작 |
|---|---|
| Scene 탭 | Scene을 전환합니다. 우클릭으로 이름을 변경할 수 있습니다. |
| Canvas Builder 탭 | UI 배치와 수정 화면을 엽니다. 기존 위치에는 `캔버스 빌더 열기` 버튼이 표시됩니다. |
| Settings 탭 | 설정 화면을 Scene 탭과 같은 위치에서 엽니다. |
| Function 탭 | 편집 중인 함수 그래프를 표시합니다. |

모든 탭은 닫기 버튼을 제공하며, 탭끼리 위치를 옮길 수 있습니다. 단, 기본 작업 흐름을 보호하기 위해 실제 프로젝트 데이터 삭제는 별도 삭제 동작에서 처리됩니다.

## Canvas Builder

Canvas Builder는 실제 앱 화면을 구성하는 공간입니다.

- Text를 추가하면 텍스트 출력 계열 노드와 연결됩니다.
- Image를 추가하면 이미지 출력 계열 노드와 연결됩니다.
- Input을 추가하면 사용자가 값을 입력할 수 있는 입력 UI와 노드 값이 연결됩니다.
- Button, Container, Vector, Custom Button을 배치할 수 있습니다.
- UI를 수정하면 연결된 노드 값 또는 Ref Key가 함께 갱신됩니다.
- UI를 선택하고 `P` 키를 누르면 연결된 노드 위치로 이동할 수 있습니다.
- 반응형 미리보기는 Desktop, Tablet, Mobile 모드를 제공하며, V1.1.3부터 가로와 세로 크기를 조절할 수 있습니다.
- Canvas Builder에서 UI 선택 후 속성 수정이 되지 않던 문제가 수정되었습니다.

## Scene Manager

Scene은 여러 화면을 가진 앱을 만들기 위한 기본 단위입니다.

- Scene 탭으로 화면을 전환합니다.
- 새 Scene을 만들면 독립된 작업 공간으로 시작합니다.
- 노드, UI, 지역변수는 현재 Scene 기준으로 관리됩니다.
- Scene 탭은 우클릭으로 이름을 변경할 수 있습니다.
- Scene 전환은 Go Scene 액션 또는 관련 노드 흐름에서 사용할 수 있습니다.

## Functions

함수는 반복해서 쓰는 노드 묶음을 직접 정의하는 기능입니다.

- 함수 만들기 버튼으로 함수 편집기에 들어갑니다.
- 편집 중인 함수는 브라우저형 탭으로 표시됩니다.
- 함수에는 이름, 설명, 매개변수, 기본값, 반환 Ref Key를 지정할 수 있습니다.
- 함수 호출 노드 또는 Action System의 Run Function 동작에서 사용할 수 있습니다.
- 재귀 호출은 가능하지만, 런타임 보호를 위해 호출 깊이 제한이 적용됩니다.

## Variables

V1.1.3부터 전역변수와 지역변수를 추가할 수 있습니다.

| 종류 | 설명 |
|---|---|
| 전역변수 | 모든 Scene에서 참조할 수 있는 값입니다. |
| 지역변수 | 현재 Scene에서만 사용하는 값입니다. |

변수 이름은 템플릿 문법에서 `{{variableName}}` 형태로 참조할 수 있습니다. Action System의 Set Variable 동작을 통해 변수 값을 바꾸는 흐름도 구성할 수 있습니다.

## Action System

UI 버튼과 Custom Button은 액션을 통해 앱 동작을 만들 수 있습니다.

| 액션 | 설명 |
|---|---|
| Open URL | HTTPS URL을 외부 브라우저로 엽니다. |
| Run Function | 사용자가 만든 함수를 실행합니다. |
| Set Variable | 전역변수, 지역변수 또는 입력 노드 값을 변경합니다. |
| Toggle UI | 특정 UI 요소를 표시하거나 숨깁니다. |
| Go Scene | 다른 Scene으로 이동합니다. |
| Play Sound | 지정한 사운드를 재생합니다. |
| Request HTTPS | HTTPS 요청 노드를 실행합니다. |

네트워크 관련 액션은 네트워크 노드 동의와 URL 보안 정책을 따릅니다.

## Settings

설정은 V1.1.3부터 탭으로 열리며, 다음 항목을 제공합니다.

| 항목 | 설명 |
|---|---|
| 언어 | 앱 UI 언어를 변경합니다. |
| 테마 | 기본 테마 또는 사용자 테마를 적용합니다. |
| 고급 모드 | 단순화된 UI 대신 더 자세한 편집 UI를 표시합니다. |
| 로그 표시 | Error Log Console 표시 여부를 설정합니다. 기본값은 꺼짐입니다. |
| 노드 연결점 크기 | 노드 연결 핸들의 크기를 조절합니다. |
| 네트워크 노드 | 네트워크 계열 노드 사용 여부를 설정합니다. |
| 동의서 | 네트워크 노드 이용 동의서를 다시 확인하고 동의를 취소할 수 있습니다. |
| 내보내기 기본값 | export 창 제목, 크기, 배경색, 리사이즈 여부 등을 설정합니다. |
| 업데이트 | GitHub Release 기준 최신 버전을 확인하고 업데이트 적용 흐름으로 이동합니다. |
| 초기화 및 복구 | 전체 초기화, 저장사항 초기화, 설정 초기화, 앱 완전 초기화를 실행합니다. |

## 보안 정책 요약

- 네트워크 계열 노드는 첫 사용 전에 동의서를 표시합니다.
- HTTPS URL만 허용합니다.
- URL 파싱 실패, localhost, loopback, 사설망 주소는 차단합니다.
- 외부 요청은 사용자 승인과 타임아웃 정책을 따릅니다.
- 중복 in-flight 요청을 방지합니다.
- 로그에는 민감할 수 있는 URL 정보를 마스킹합니다.
- 브라우저 열기와 외부 링크도 동일한 URL 정책을 적용합니다.
- 스크립트 노드는 기본적으로 제한 실행 모드를 사용하며, 전체 JavaScript 모드는 프로젝트 신뢰와 강한 경고 UI를 요구합니다.
- 파일 감시 노드는 별도 사용자 승인을 요구합니다.

## 내보내기

### Desktop Pipeline

| 대상 | 설명 |
|---|---|
| Windows `.exe` | Windows 즉시 실행형 산출물을 만듭니다. |
| Linux bundle / `.zip` | Linux 실행 파일과 필요한 리소스를 포함한 번들을 만듭니다. |

export 창에서는 앱 이름, 아이콘, 저장 경로, 출력 형식을 지정해야 합니다. 지정하지 않은 경우 기본 이름과 기본 아이콘이 사용됩니다.

### Mobile Pipeline

모바일 export는 데스크톱 산출물과 분리된 워크스페이스로 생성됩니다.

| 대상 | 설명 |
|---|---|
| Android `.apk` pipeline | Android 패키징을 위한 모바일 워크스페이스를 생성합니다. |
| iOS `.ipa` pipeline | iOS 패키징을 위한 모바일 워크스페이스를 생성합니다. |

모바일 export는 `web/`, `project.json`, `mobile-export.json`, `package.json`, `capacitor.config.json`, 플랫폼별 아이콘 세트를 포함합니다.
