# IXO Engine

IXO Engine은 노드를 연결해 데스크톱 애플리케이션의 로직과 화면을 함께 설계하는 비주얼 앱 빌더입니다. 사용자는 JavaScript를 직접 쓰지 않아도 입력, 조건, 계산, 화면 출력, 네트워크, 사운드, 반복 흐름을 조합해 앱을 만들 수 있고, 필요할 때는 스크립트 노드와 사용자 정의 함수로 더 깊은 동작까지 확장할 수 있습니다.

현재 문서는 V1.1.1 기준입니다.

## 핵심 철학

- 로직은 노드 그래프로 표현합니다.
- 화면은 Canvas Builder에서 시각적으로 설계합니다.
- UI 요소와 노드는 1:1로 연결되어 실시간으로 동기화됩니다.
- exported runtime은 에디터와 같은 런타임 코드를 사용해 결과 차이를 최소화합니다.
- 프로젝트 데이터는 로컬 우선으로 보관되며, 손상 시 Safe Mode로 복구합니다.

## 주요 작업 영역

| 영역 | 설명 |
|---|---|
| Node Workspace | 노드를 배치하고 연결해 실행 흐름을 만듭니다. |
| UI Viewer | 완성된 UI를 사용자 시점으로 확인합니다. |
| Canvas Builder | 텍스트, 이미지, 버튼, 컨테이너를 직접 배치하고 수정합니다. |
| Inspector | 선택한 노드나 UI 요소의 세부 속성을 편집합니다. |
| Functions | 사용자 정의 함수를 만들고, 별도 함수 그래프 안에 노드를 배치합니다. |
| Scene Manager | 왼쪽 사이드바에서 화면(Scene)을 버튼으로 전환, 생성, 정리합니다. |
| Asset Manager | 이미지, 사운드, `.ixo` 프로젝트 파일, 테마 파일을 관리합니다. |
| Error Log Console | 실행 결과와 외부 동작 로그를 확인합니다. |

## Canvas Builder

Canvas Builder는 화면 설계를 담당합니다.

- 텍스트를 추가하면 `텍스트 출력` 노드가 자동 생성됩니다.
- 이미지를 추가하면 `이미지 출력` 노드가 자동 생성됩니다.
- 버튼과 컨테이너도 대응하는 기존 표준 노드로 자동 편성됩니다.
- UI를 수정하면 연결된 노드 값이 실시간으로 갱신됩니다.
- UI를 선택하고 `P` 키를 누르면 해당 워크스페이스 노드로 즉시 이동합니다.
- 노드에서 값을 바꾸면 연결된 UI에도 반영됩니다.
- UI는 마우스로 이동과 크기 조절을 할 수 있습니다.
- Vector 요소와 Custom Button은 Inspector의 Vector Draw Pad에서 직접 그리듯이 편집할 수 있습니다.

## Scene Manager

V1.1.1부터 Scene은 문자열 입력보다 명확한 버튼 기반 UI로 관리합니다.

- 왼쪽 사이드바의 `Scenes` 영역에서 Scene을 바로 전환합니다.
- `+ Scene` 버튼으로 새 화면을 만듭니다.
- Canvas Builder에서 새로 추가하는 UI는 현재 선택한 Scene에 들어갑니다.
- Scene을 정리할 때는 해당 Scene의 UI를 `main`으로 이동해 데이터 손실을 줄입니다.

## Asset Manager

Asset Manager는 제작에 필요한 파일을 프로젝트 안에서 관리합니다.

| 파일 종류 | 동작 |
|---|---|
| 이미지 파일 (`.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`, `.svg`) | Canvas Builder로 끌어 놓으면 이미지 UI와 연결 노드가 자동 생성됩니다. |
| `.ixo` 파일 | Node Workspace 또는 Canvas Builder로 끌어 놓으면 포함된 노드와 UI를 현재 프로젝트에 붙여넣습니다. |
| 오디오 파일 | 사운드 노드 또는 배경음 관련 흐름에 사용할 수 있습니다. |
| 테마 JSON / `.ixo-theme` | 설정에서 업로드해 IXO Engine 편집기 UI 테마로 사용할 수 있습니다. |

## Theme Presets와 사용자 테마

테마는 편집기 전체 UI의 색상, 패널, 입력창, 경계선, 강조색에 적용됩니다. export한 산출물의 앱 디자인은 프로젝트 UI와 export 설정을 따르며, 편집기 테마 변경만으로 산출물 UI가 강제로 바뀌지는 않습니다.

- 기본 프리셋: Mint Console, Glass Panel, Pixel Retro, Clean SaaS, Game HUD, Dark Red, Ocean Blue
- 사용자 지정 테마: 설정에서 JSON 또는 `.ixo-theme` 파일 업로드
- 테마 적용 범위: 사이드바, 워크스페이스, Inspector, 버튼, 입력창, 로그 패널 등 편집기 전체

## 사용자 정의 함수

IXO Engine의 함수는 사용자가 직접 만드는 별도 그래프입니다.

- `함수 만들기`를 눌러 함수 편집기로 진입합니다.
- 함수 안에는 일반 노드와 함수 호출 노드를 모두 넣을 수 있습니다.
- 재귀 함수도 허용되며, 런타임은 무한 재귀를 막기 위해 호출 깊이 제한을 둡니다.
- 각 함수는 다음 정보를 가질 수 있습니다.

| 필드 | 설명 |
|---|---|
| 이름 | 함수 호출 노드에 표시될 이름입니다. |
| 설명 | 함수가 무엇을 하는지 적는 문서용 설명입니다. |
| 매개변수 | 이름, 기본값, 설명을 각각 가질 수 있습니다. |
| 반환 Ref Key | 함수 실행 뒤 결과로 사용할 컨텍스트 키입니다. |

예를 들어 `calculateScore(scoreA, scoreB)` 함수를 만들고 반환 Ref Key를 `totalScore`로 지정하면, 함수 호출 노드는 내부 그래프에서 계산한 `totalScore` 값을 결과로 돌려줍니다.

## Safe Mode와 데이터 복구

클라이언트 데이터가 꼬였을 때를 대비해 Safe Mode가 준비되어 있습니다.

- 자동 저장 데이터 검증
- 마지막 백업 저장본 복원
- 마이그레이션 코드 기반 구버전 프로젝트 보정
- 복구 실패 시 안전한 기본 프로젝트로 자동 초기화
- 수동 백업 복원과 수동 초기화 버튼 제공
- 단일 인스턴스 실행으로 중복 실행에 의한 충돌 가능성 감소

## 보안

네트워크 계열 노드는 기본적으로 보수적으로 동작합니다.

- 앱 최초 실행 시 네트워크 계열 노드 사용 동의를 묻습니다.
- 설정에서 이후에 다시 켜거나 끌 수 있습니다.
- HTTPS만 허용합니다.
- URL 파싱 실패, localhost, 사설망 주소는 차단합니다.
- 첫 외부 요청은 사용자 승인을 요구합니다.
- 요청에는 타임아웃과 중복 실행 방지가 적용됩니다.
- 로그에는 민감한 URL 정보가 마스킹됩니다.
- 브라우저 열기 노드도 같은 보안 정책을 따릅니다.

## 업데이트 확인과 개인정보

- 앱은 시작 시 한 번만 GitHub Release를 확인합니다.
- 업데이트 확인은 GitHub API와 직접 통신하며, 프로젝트 내용은 민양테크 서버로 자동 업로드되지 않습니다.
- 사용자가 직접 배치한 네트워크 노드와 브라우저 열기 노드는 사용자가 지정한 외부 서버와 통신할 수 있습니다.
- 자세한 내용은 [개인정보 처리방침](/docs/ixo/privacy.html)에서 확인할 수 있습니다.

## 내보내기

### Desktop Pipeline

| 대상 | 설명 |
|---|---|
| `.exe` | Windows 즉시 실행형 산출물 |
| Linux bundle | Linux 실행 파일과 필요한 리소스 |

### Mobile Pipeline

모바일은 데스크톱 산출물과 별도 파이프라인으로 처리합니다.

| 대상 | 설명 |
|---|---|
| `.apk` pipeline | Android 패키징 워크스페이스 생성 |
| `.ipa` pipeline | iOS 패키징 워크스페이스 생성 |

모바일 export는 exported runtime과 프로젝트 데이터를 `web/`, `project.json`, `mobile-export.json` 형태로 묶은 전용 워크스페이스를 만듭니다. 여기에 `package.json`, `capacitor.config.json`, 점검 스크립트까지 함께 생성되어 곧바로 Capacitor 기반 네이티브 래퍼 흐름으로 이어갈 수 있습니다.

```text
npm install
npm run mobile:check
npm run mobile:add
npm run mobile:sync
npm run mobile:open
```

Android 최종 빌드는 Android SDK가 준비된 환경에서, iOS 최종 빌드는 macOS와 Xcode 환경에서 진행합니다.

## 권장 사용 흐름

1. `시작점` 노드에서 흐름을 시작합니다.
2. 입력, 계산, 조건 노드를 연결합니다.
3. Canvas Builder에서 화면을 만듭니다.
4. UI와 노드가 맞물리는지 Viewer에서 확인합니다.
5. 필요한 로직은 사용자 정의 함수로 분리합니다.
6. Desktop 또는 Mobile export 파이프라인을 선택해 산출물을 만듭니다.
