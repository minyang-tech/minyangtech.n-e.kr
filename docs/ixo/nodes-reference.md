# IXO Engine 노드 상세 문서

## 공통 개념

| 항목 | 설명 |
|---|---|
| `Label` | 노드의 표시 이름입니다. |
| `Value` | 노드가 계산하거나 전달할 기본 값입니다. |
| `Ref Key` | 다른 노드가 `{{refKey}}` 형식으로 참조할 수 있는 변수 이름입니다. |
| 템플릿 문법 | `{{username}}`, `{{score}}`처럼 이전 노드의 출력값을 문자열 안에 넣을 수 있습니다. |
| 실행 흐름 | 연결선 방향에 따라 위상 순서로 실행됩니다. |
| 출력값 | 대부분의 노드는 결과를 `context[refKey]`에 저장합니다. |
| 조건식 | `==`, `!=`, `>`, `<`, `>=`, `<=`, `AND`, `OR`를 지원합니다. |
| 상태 표기 | 아래 표의 `구현 상태`는 현재 코드 기준입니다. |

## 구현 상태 기준

| 상태 | 의미 |
|---|---|
| 구현됨 | 현재 런타임에서 실제 계산, 분기, 표시, 요청, 재생 등이 동작합니다. |

# 1. Core 노드

## 시작점

| 항목 | 내용 |
|---|---|
| 역할 | 플로우의 시작 위치를 나타냅니다. 연결된 다음 노드들의 실행을 시작합니다. |
| 입력 | 없음 |
| 출력 | 일반적으로 빈 값 또는 시작 컨텍스트 |
| 사용 예 | 앱이 시작되면 `시작점 -> 입력 칸 -> 조건 분기` 순서로 실행 |
| 구현 상태 | 구현됨 |

## 조건 분기

| 항목 | 내용 |
|---|---|
| 역할 | 조건식을 평가해 참이면 `true`, 거짓이면 `false` 경로로 흐름을 나눕니다. |
| 입력 | 조건식 |
| 출력 | `"true"` 또는 `"false"` |
| 입력 예 | `{{score}} >= 60 AND {{name}} != ""` |
| 사용 예 | 점수가 60 이상이면 합격 흐름, 아니면 불합격 흐름으로 분기 |
| 구현 상태 | 구현됨 |

## 값 비교

| 항목 | 내용 |
|---|---|
| 역할 | 두 값의 비교를 표현하기 위한 노드입니다. |
| 입력 | 비교식 문자열 |
| 출력 | 현재 구현에서는 템플릿이 해석된 문자열 |
| 입력 예 | `{{a}} == {{b}}` |
| 사용 예 | 조건 분기 앞에서 비교식을 구성하는 용도 |
| 구현 상태 | 구현됨 |
| 비고 | 이름은 비교 노드이지만, 현재 런타임에서는 자체적으로 참/거짓을 계산하지 않고 문자열 전달 중심입니다. |

## 데이터 합치기

| 항목 | 내용 |
|---|---|
| 역할 | 여러 값을 하나의 문자열로 결합합니다. |
| 입력 | 템플릿 문자열 |
| 출력 | 결합된 문자열 |
| 입력 예 | `{{firstName}} {{lastName}}` |
| 사용 예 | 이름 조합, 주소 조합, 메시지 포맷 생성 |
| 구현 상태 | 구현됨 |

## 스크립트

| 항목 | 내용 |
|---|---|
| 역할 | 기본적으로 제한 실행 모드에서 안전한 단일 `return` 식을 실행하고, 필요 시 사용자 승인 후 전체 JavaScript 모드로 전환합니다. |
| 입력 | JavaScript 코드 |
| 출력 | `return`으로 반환한 값 |
| 입력 예 | `return context.username || "guest";` |
| 사용 예 | 복잡한 계산, 조건 처리, 문자열 변환 |
| 구현 상태 | 구현됨 |
| 비고 | 제한 모드에서는 `context`, `inputValues`를 읽는 단일 반환식만 바로 실행됩니다. 함수 호출이나 여러 문장은 전체 JavaScript 승인 전까지 차단됩니다. |

| 횟수 반복 | 지정한 횟수만큼 반복 실행을 표현합니다. | `10` | 구현됨 |

| 반복 끝내기 | 현재 반복문을 종료합니다. | `break` | 구현됨 |

| 복제본 삭제 | 생성된 복제 인스턴스를 제거하는 의미의 노드입니다. | `clone` | 구현됨 |

# 7. Logic 노드

| 노드 | 역할 | 입력 예 | 구현 상태 |
|---|---|---|---|
| 수식 계산 | 간단한 사칙연산을 계산합니다. | `{{a}} + {{b}}` | 구현됨 |
| 마우스 눌림? | 포인터 입력 상태를 조건으로 표현합니다. | `false` | 구현됨 |

| 포인터 닿음? | 포인터가 특정 대상 위에 있는지를 표현합니다. | `false` | 구현됨 |

| RGB를 HEX로 | RGB 숫자 값을 HEX 색상 코드로 변환합니다. | `255, 0, 0` | 구현됨 |
| HEX 채널 | HEX 색상값에서 특정 채널을 다룹니다. | `#ff0000 R` | 구현됨 |
