# IXO Engine 개인정보 처리방침

시행일: 2026-05-18

민양테크는 IXO Engine 이용 과정에서 처리될 수 있는 정보의 범위와 이용자의 권리를 명확히 안내하기 위해 본 개인정보 처리방침을 공개합니다. IXO Engine은 로컬 우선 방식으로 동작하는 비주얼 앱 빌더이며, 프로젝트 데이터는 기본적으로 이용자의 장치에 저장됩니다.

## 1. 개인정보의 처리 목적

IXO Engine은 다음 목적을 위해 필요한 범위에서만 통신을 수행합니다.

| 구분 | 목적 |
|---|---|
| 업데이트 확인 | 앱 시작 시 1회 최신 버전 여부 확인 |
| 네트워크 계열 노드 | 이용자가 직접 구성한 `HTTPS 요청`, `브라우저 열기` 기능 실행 |

프로젝트 파일, 자동 저장 데이터, Canvas Builder 구성 정보는 민양테크 서버로 자동 업로드되지 않습니다.

## 2. 처리될 수 있는 정보 항목

| 구분 | 처리될 수 있는 정보 |
|---|---|
| 업데이트 확인 | GitHub API 연결 과정에서 발생할 수 있는 IP 주소 등 통신 메타데이터 |
| 이용자가 만든 네트워크 노드 | 이용자가 입력한 URL, 요청 본문, 헤더, 토큰, 개인정보 등 |
| 브라우저 열기 노드 | 이용자가 지정한 외부 URL |

IXO Engine 자체는 이용자의 계정 정보나 프로젝트 내용을 민양테크 서버로 자동 수집하지 않습니다. 다만 이용자가 외부 서버로 전송하도록 설계한 데이터는 해당 서버 운영자에게 전달될 수 있습니다.

## 3. 개인정보의 처리 및 보유 기간

- 민양테크는 업데이트 확인과 관련하여 별도 서버에 개인정보를 저장하지 않습니다.
- 로컬 프로젝트와 자동 저장 데이터는 이용자가 삭제하거나 초기화할 때까지 이용자의 장치에 남을 수 있습니다.
- 이용자가 외부 서버로 전송한 정보의 보유 기간은 해당 외부 서비스 운영자의 정책을 따릅니다.

## 4. 제3자 제공, 처리위탁 및 국외 이전

- 민양테크는 IXO Engine을 통해 수집한 개인정보를 별도 제3자에게 제공하거나 처리위탁하지 않습니다.
- 업데이트 확인에는 GitHub API가 사용됩니다.
- 이용자가 만든 네트워크 노드가 호출하는 외부 서비스는 민양테크가 운영하지 않습니다.
- 이용자가 선택한 외부 서비스가 국외에 있을 수 있으므로, 해당 서비스의 개인정보 처리방침과 보안 수준은 이용자가 별도로 확인해야 합니다.

## 5. 정보주체의 권리와 행사 방법

- 로컬에 저장된 프로젝트와 자동 저장 데이터는 앱 기능을 통해 삭제 또는 초기화할 수 있습니다.
- 외부 서비스로 직접 전송한 정보의 열람, 정정, 삭제, 처리정지 요구는 해당 서비스 운영자에게 행사해야 합니다.

## 6. 파기 절차 및 방법

- 로컬 프로젝트와 자동 저장 데이터는 이용자가 삭제 또는 초기화 기능을 실행할 때 제거됩니다.
- 민양테크 서버에 별도 저장되는 프로젝트 데이터는 없습니다.

## 7. 안전성 확보 조치

IXO Engine은 네트워크 계열 노드에 다음 보호 조치를 적용합니다.

- HTTPS 강제
- URL 파싱 실패 차단
- localhost 및 사설망 주소 차단
- 첫 외부 요청 전 이용자 승인
- 타임아웃 및 중복 요청 방지
- 로그 내 URL 마스킹
- 브라우저 열기 노드와 외부 링크에도 동일 정책 적용

## 8. 이용자 유의사항

- 비밀번호, 인증 토큰, 주민등록번호, 결제 정보 등 민감한 정보는 신뢰할 수 없는 서버나 스크립트에 입력하지 마십시오.
- 이용자가 직접 입력하거나 연결한 URL, 서버, 데이터의 적정성은 이용자가 확인해야 합니다.
- 외부 서비스의 내용, 보안, 개인정보 처리 방식은 민양테크가 통제할 수 없습니다.
